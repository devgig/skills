# Constitution — C# Microservice (platform level)

> A **constitution** is the set of non-negotiable principles every service
> scaffolded by `scaffold-csharp-microservice` obeys. It's separate from any
> individual service's `spec.md` because it applies to *all* services built
> from this skill. Edit this file to change the house style; edit `spec.md`
> to change one service.
>
> Concept adopted from [GitHub Spec Kit](https://github.com/github/spec-kit) —
> see its `templates/memory/constitution.md` for the canonical form. This
> constitution is narrower: it governs C# microservices specifically.

---

## Language and runtime

- **.NET 8**, C# 12. No multi-targeting.
- `Nullable` and `ImplicitUsings` enabled on every project.

## Solution layout — fixed

Every service has exactly these six projects:

```
src/
  Service/         ASP.NET Core minimal-API host
  Application/     MediatR commands, queries, handlers
  Domain/          Entities + repository interfaces
  Infrastructure/  Generic IRepository<T> + BaseRepository<T>
tests/
  UnitTests/
  IntegrationTests/
```

No `Consumer/` project by default. Do not add projects not on this list
without amending this constitution.

## Architectural patterns — mandatory

- **CQRS via MediatR.** State-changing requests are `Command`s, read-only
  requests are `Query`s. Handlers are `sealed class : IRequestHandler<,>` and
  live in `Application/Handlers/`.
- **Repository pattern via `BaseRepository<T>`.** All persistence inherits the
  generic base; direct `IMongoCollection<T>` usage outside the repository is
  prohibited.
- **Minimal APIs.** Controllers are not used.
- **FluentValidation** for request validation, invoked inline in each
  endpoint lambda — validation failures throw `ValidationException`, caught
  by the `GlobalExceptionHandler`.
- **RFC 7807 ProblemDetails** for all error responses, produced by
  `GlobalExceptionHandler`.
- **DTOs are records**, not classes.

## Persistence

- **MongoDB** is the only supported database. Postgres/SQL Server variants
  would require a separate skill.
- Entity `Id` is `string` with
  `[BsonId][BsonRepresentation(BsonType.ObjectId)]` unless the `spec.md`
  explicitly says otherwise.
- Indexes live in the repository constructor (`_collection.Indexes.CreateOne`).

## Observability — mandatory

- **Serilog** structured logging with Console + rolling File sinks. Enriched
  with MachineName, ProcessId, ThreadId, FromLogContext.
- **OpenTelemetry** traces + metrics. Traces export via OTLP/gRPC; metrics
  export via Prometheus scrape endpoint at `/metrics`.
- `ErrorAwareSampler` forces sampling on spans tagged with errors or
  exceptions.
- `MongoDB.Driver.Core.Extensions.DiagnosticSources` is registered as a span
  source.

## Testing — mandatory

- **xUnit** is the only test framework.
- `Moq` + `FluentAssertions` for unit tests.
- `EphemeralMongo` for integration tests — no external MongoDB dependency at
  test time.
- Tests the user supplies via `tests-inbox/` or the spec's **Tests** section
  are **the acceptance contract.** The skill will not modify them to make
  them pass.

## What the skill MUST NOT do

- Invent fields, endpoints, types, routes, validation rules, or business
  rules the spec doesn't declare.
- Add new projects, abstractions, or NuGet packages not listed here without
  an explicit request.
- Include authentication, authorization, or auditing — those are
  service-specific and must be added post-scaffold.
- Ship with an insecure default (wide-open CORS, plaintext secrets, etc.).
  Better to ship with nothing than with an unsafe default.

## How to change the constitution

This file is the platform-level contract. Amending it affects every future
service generated by this skill.

1. Open a PR that edits `constitution.md`.
2. Update any template files or SKILL.md prose that contradict the change.
3. Regenerate one reference service (UserService) to verify the change
   produces a buildable, testable output.
4. Merge.

Services generated before a constitutional change are not retroactively
updated unless their spec is re-run.
